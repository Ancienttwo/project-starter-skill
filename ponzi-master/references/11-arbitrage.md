# 套利框架方法论（Arbitrage Framework）

> 来源：Open Rug 开源镰刀系列 - 26
> 核心观点："除非你是乔布斯或黄仁勋，否则你能做的只是套利"

---

## 一、套利的本质

### 1.1 万物皆套利

> "这个地球上绝大多数伟大的'商业创新'，包括Crypto，都是基于套利"

| 行业 | 套利类型 |
|------|----------|
| 贸易 | 地理套利 |
| 银行 | 对储户的认知套利 |
| 北印度互联网 | 防火墙的监管套利 |
| Crypto | 全球监管套利 |

### 1.2 套利的定义

**广义套利**：在一个场景内，根据不改变的规则，利用场景内各参与方"不对称"的权责或行为能力，以不成比例的更小成本获利的行为。

## 二、套利三要素分析框架

### 2.1 三个核心问题

| 要素 | 问题 |
|------|------|
| **套利来源** | 我套谁的利？ |
| **规则与不对称** | 套的什么利？ |
| **套利空间** | 不对称有多大？ |

### 2.2 浅层套利案例：Hyperliquid

| 要素 | 分析 |
|------|------|
| **套利来源** | 拿牌照差价合约平台 + 大离岸CEX因合规服务不了 + 信不过小所的交易员贡献的手续费 |
| **规则和不对称** | 币圈合约需求确定，但能提供服务的CEX要么不被信任，要么受监管制约 |
| **套利空间** | 全球币圈合约需求 - 被合规束缚的CEX能提供的交易量 |

**核心洞察**：
> Hyperliquid的套利核心是把中心化合约所包装成一条链，形成"监管套利"——"行CEX之实而不接受CEX同等的监管"

### 2.3 深层套利案例：博彩

| 层次 | 理解 |
|------|------|
| **表面** | 靠凯利公式，赚赌徒认知差 |
| **实际** | 需要洗白的钱冲进来当赌资，左手输给右手，黑钱进白钱出 |

**后者比前者大百倍**

| 要素 | 分析 |
|------|------|
| **套利来源** | 有需求走赌场过一遍的流水付出的磨损和抽水 |
| **规则和不对称** | 其他价格双轨制行业因监管无法容纳类似需求 |
| **套利空间** | 该需求流水总量 - 其他双轨制行业可处理的流水 |

**核心策略**：为此类流水合理化入金做好监管套利，而不是仅仅为赌客合规

> **反例**：澳洲星城赌场不转型线上反而扩张房地产，政府一严卡反洗钱直接破产

## 三、套利即叙事

### 3.1 套利是最好的故事

> "套利逻辑不仅是好业务，也是好故事。套利场景和空间越大，叙事越值钱"

**原因**：连门外汉都能懂，觉得可持续风险小，是值得参与的机会

### 3.2 币圈历史最牛叙事

| 叙事 | 套利类型 |
|------|----------|
| BTC挖矿 | 能源套利 |
| Plustoken | "搬砖套利"（骗局叙事） |
| USDE | 策略中性套利 |
| Binance Alpha | 撸毛套利 |

> "最易懂最自洽的故事最好传播，最好传播的故事才是最好的故事，最好的故事才能套最多的人"

## 四、针对流动性的套利

### 4.1 核心原则

> "币圈只有两种稀缺资源：流动性和屏幕时间"

同一套利场景，在同一时空中，其空间（策略容量）必定有限。**占领绝对份额 = 吸引最大流动性**

### 4.2 杠杆放大策略

> "一旦找到套利空间，就必须10倍100倍上杠杆放大核心策略，最快速度占据套利空间"

**稳定币杠杆叠加公式**：

```
原始策略容量：1份
↓
发成稳定币，找借贷协议当抵押物（LTV 60%）→ 3x
↓
Pendle PT/YT → 4x
↓
当资产发行拆分盘对价或嵌套成JLP + 贴息 → 10x
↓
最终杠杆：100x
```

> "蚂蚁金服在传统都敢玩儿，币圈玩这个都算邯郸学步"

### 4.3 USDE案例

**为什么干一年就把赛道堵死了？**

- 在其他借贷协议上了大量杠杆
- 把套利空间中能提供的流动性都占据了
- 后来者无空间可套

## 五、针对屏幕时间的套利

### 5.1 屏幕时间的价值

人一天清醒时间最多17小时，包括：
- 为赚钱不得不占用的时间
- 多巴胺驱动的娱乐时长

**最大化屏幕时间的方法**：赚钱和多巴胺必须合二为一

### 5.2 POW的低效性

既然时间有限，任何POW都是低效的（最大利润固定）。

**唯一解**：博弈，而且是最大杠杆的博弈

### 5.3 传统平台的尝试

| 平台 | 方式 | 限制 |
|------|------|------|
| Tiktok | 越南主播 | 监管限制 |
| 斗鱼 | 军火商/工会返点 | 政策打击 |
| 短视频/直播 | 冲流水 | 无法与赌直接结合 |

### 5.4 Pump Fun的套利空间

**公式**：
```
这一代人被多巴胺控制偏移"理性人"的值 - 老登娱乐平台吃了的 = Pump Fun套利空间
```

**为什么挑战Pump的都失败了？**

> "拿'更好的公平发射Launchpad'叙事挑战Pump的都没看懂"

Pump做的是：
- 把大众社交媒体变成资产发行的荷官
- 每一分钟屏幕时长都变成100倍杠杆的发射
- 打狗人把刷推时间都用在验CA上
- **娱乐时间 + 100倍杠杆 + 赚钱时间 = 三合一**

### 5.5 Sidekick模式

**Pump的反面，同样占领用户时间**：

| 特点 | 说明 |
|------|------|
| 效率 | 一小时直播集中发50个CA角度 |
| 省时 | 连翻Twitter验证时间都省了 |
| 转型 | 斗鱼事件中的军火商可直接转型链上狗庄 |
| 变现 | 前店后厂+礼物打赏，流水更多 |

> "现在的1sol挑战主播，可能都是未来的'小团团'"

## 六、套利策略设计模板

### 6.1 识别套利机会

```
步骤1：我套谁的利？
- 谁在为这个场景付费？
- 谁的行为能力受限？

步骤2：套的什么利？
- 规则是什么？规则不变吗？
- 不对称在哪里？

步骤3：套利空间多大？
- 总需求是多少？
- 能被满足的需求是多少？
- 差值 = 套利空间
```

### 6.2 设计套利策略

```
步骤4：如何占据套利空间？
- 能上多少杠杆？
- 如何最快占据空间？

步骤5：如何包装叙事？
- 这个套利逻辑门外汉能懂吗？
- 如何让它看起来可持续？
```

### 6.3 套利检查清单

| 检查项 | 问题 |
|--------|------|
| 来源清晰 | 我能明确说出套谁的利吗？ |
| 不对称存在 | 这个不对称是结构性的还是临时的？ |
| 空间足够 | 套利空间能支撑我的目标吗？ |
| 杠杆可用 | 我能用什么方式放大杠杆？ |
| 叙事可传播 | 我能用一句话解释清楚吗？ |

## 八、总结
