# 互助盘详解（Mutual Aid Ponzi）

> 来源：Open Rug 开源镰刀系列 - 三盘理论2
> 原文发布：2024年1月4日

---

## 一、定义与本质

### 1.1 基本定义

**互助盘**是指：以资金拆借为形式，通过给拆借资金提供收益，错配资金为目的的资金盘。

> "A欠B一百，B欠C一百，C欠D一百，这时候D拿了一百去找A洗了个脚，ABCD的帐就都还上了。"
>
> "这个笑话你肯定听过，但如果这一切发生在一天内, APY 696969%呢？"

**核心目标**：错配资金规模越大，时间越长，庄越赚钱。

### 1.2 特征矩阵

| 特征维度 | 属性 | 说明 |
|----------|------|------|
| 资金池 | ❌ 否 | 无中心化资金池 |
| 自由进出 | ❌ 多数否 | 按规则出金 |
| 固定收益 | ✅ 是 | 名义上无泡沫，承诺固定收益 |

### 1.3 传统金融中的互助盘

**典型案例**：银行、P2P、债券

> "银行和P2P通过利息来吸储形成资金池，根本没有100%的钱在金库(银行称储备金制度), 而是把钱放出去赚利差。只要永远有别人存进来的钱大于提现金额，这个盘子就能维持,反之就遇到挤兑危机。"

**历史案例**：
- 硅谷银行破产事件
- 中国P2P爆雷

> "本质是中心化资金池导致的崩盘"

## 二、庄家视角

### 2.1 盈利方式

互助盘庄家主要通过两种方式获利：

| 方式 | 定义 | 特点 |
|------|------|------|
| **抽水** | 在整个互助的资金错配中，收取流水的一个百分比作为"费用" | 持续性收入 |
| **清算** | 在盘子无法继续时触发规则，直接从盘子剩余资金中一次性抽走一个百分比 | 一次性收入 |

### 2.2 盈利公式

```
profit(互助盘) = 总互助状态中资金 × %老鼠仓账户比例
```

### 2.3 崩盘逻辑

**传统崩盘模型**：
1. 资金池被挪用，直接关盘，rug池子跑路
2. 到期无法兑付，进入清算阶段（如FTX）

**Crypto互助盘崩盘条件**：
- 全局债务 > 可清算资产 + 外部流动性
- 出金秩序失控，留出套利空间

---

## 三、Crypto对互助盘的革命

### 3.1 智能合约的意义

> "Crypto尤其智能合约之于互助盘就好比蒸汽机之于工业革命，它从技术上消除了中心化资金池的风险"

### 3.2 互助盘的核心问题

互助盘要做大规模，必须解决：
- 如何实现资产发行（铸币权）
- 如何控制出金秩序（避免套利）
- 如何设置清算门限（避免无限债务扩张）

## 四、互助盘模型分类

### 4.1 纯动态互助模型

**代表**：3M、Forsage

**规则**：
- 根据系统排单，用银行账户给指定账户按指令转钱
- 获得相应收益（来自之后其他人被排单打给你）
- 收益按推广情况+基础每笔收益计算

**Forsage变体**：没有排单，后来人打ETH直接分配给上家

**特色**：
- ❌ 没有静态收益，纯靠拉新动态奖励
- ❌ 没有资金池

**庄家获利模型**：变相抽水
- 3M：每天排单中留一个%将接收账户变成庄家账户
- Forsage：庄家占据有利点位，通过后续玩家入金中的"进贡"抽水

**崩盘模型**：项目随入金/打款停止而停止，无清算

> "Forsage作为18年老盘子直到今天仍运行"

**总结评价**：
| 维度 | 评价 |
|------|------|
| 准备金 | 无 |
| 拨比泡沫 | 无 |
| 市场成本 | 极高（相当于纯CX） |
| 适用场景 | 空投模型活动冷启动 |
| 可组合性 | 高，可被组合进复杂互助盘 |

---

### 4.2 类期权模型

**代表**：比特币超3、FOMO3D、泰山众筹

**定义**：一个聚集资金的互助池用新进资金给参与者分红或获利退出，在达到某种结束条件时触发清算重启。

#### 4.2.1 FOMO3D

**规则概要**：
- 以不断上涨的入场价格买池子的债券
- 能分多久看什么时候没人入金
- 总资金划出一部分做抽奖和尾单博弈
- 大家为了拿奖和博最后一棒分成不断买入

**设计目的**：最大化资金规模，拉长时间

**庄家获利**：占早期点位 + 团队2%分润

**清算机制**：基本为本金100%清算（可能本金全损）

#### 4.2.2 比特币超3/泰山众筹

**与FOMO3D的区别**：
- 获利有上限
- 最后两轮只损失一个百分比
- 重开优先进入
- 损失的%归池子（项目方）

**总结评价**：
| 维度 | 评价 |
|------|------|
| 准备金 | 无 |
| 拨比泡沫 | 无 |
| 市场成本 | 较低，易fomo与传播 |
| 局限性 | 纯粹分钱，很难融入资产发行 |

> "发行=铸币，铸币需要泡沫"

---

### 4.3 众筹池模型（资产发行型互助盘）

**代表**：流动性挖矿、OHM、MoneyArk

#### 4.3.1 资产发行的意义

> "天使投资分币比分股东线道德风险低"
> "资产发行也就是拥有铸币权，可以像政府一样'纸钞换黄金'，扩大营收比"

#### 4.3.2 流动性挖矿/AMM公平发射

**创新**：解决了ICO时代"钱都到庄手里"的问题

> "钱在去中心化池子里，币也在池子里"
> "相当于LP拿钱从池子买了币然后做了个融资租赁，把币和钱租给池子收币本位的利息"

**问题**：
- LP不是有序的无泡沫分池子
- 可以无限挖矿且退出方式无序
- LP处于相互博弈状态
- 更像可自由退出本金的矿机分红盘
- 外部买盘不足通胀时，极易踩踏

#### 4.3.3 OHM模型

**创新**：变LP为矿机
- 一次性投入买债券
- 分币收益
- 用卖矿机收入和回购增发机制托底OHM价格

#### 4.3.4 MoneyArk模型

**核心规则**：
1. 以USDC入金，入金数量即全局债务
2. 每七天需至少复投200美金才能继续复利
3. 入金的10%作准备金兑付，85%用于"自动回购"购买$MARK
4. $MARK买卖收10%税（5%分红，5%加池子）
5. 49%代币由黑洞合约持有，超过51%时自动卖出

**设计特点**：
- 金本位入金（入场成本始终定数）
- 100%本金出局机制
- 理论上85%回购+10%买卖税覆盖全局债务即可无人亏损

**问题分析**：
> "如果我告诉你，你一入金，85%的钱就进了项目方的口袋，相当于P2P里一个中心化的资金池，用来随他们的意给$MARK拉盘呢？"

**庄家套利方式**：
1. 不断让Mark走出区间波段
2. 在相对低点买入Mark
3. 用"85%拉盘资金"在相对高点大额拉盘
4. 抛售Mark套利
5. 重复直到盘子崩

> "相当于在互助盘里放了一个看不见的抽水机制，抽水的大小时间都是随团队喜欢。这可比FOMO3D那些明着抽2%的爽多了"

## 五、互助盘与算法稳定币

### 5.1 算稳的互助盘本质

> "在crypto里面，脱胎于互助盘的最大赛道是：非超额抵押型算稳——LUNA、OHM均属于这个类别"

### 5.2 算稳的命门

因为是互助盘，命门与互助一致：
- 全局债务 > 可清算资产 + 外部流动性 → 崩盘
- 出金秩序必须可控不能留套利空间

### 5.3 LUNA崩盘教训

**问题**：无限债务扩张（缺乏清算门限）

**设计必要条件**：
1. 必须有明确的清算门限
2. 必须有严格的出金秩序
3. 资产发行泡沫增速必须与国库增速一致

> "资产是用来割晚进场的一级和二级，不是让你送钱的"

## 六、互助盘设计要点

### 6.1 核心设计原则

| 设计维度 | 要点 |
|----------|------|
| 清算门限 | 必须明确，避免无限债务扩张 |
| 出金秩序 | 必须严格，不能留套利空间 |
| 泡沫增速 | 与国库增速一致 |
| 抽水机制 | 可见或隐藏，但必须存在 |

### 6.2 互助盘特点总结

**优势**：
- 规模大
- 速度快
- 无中心化资金池风险（Crypto）

**劣势**：
- 设计和数值必须提前清晰
- 开盘后调整空间极小
- 经典互助盘难以做资产发行

### 6.3 与其他盘型结合

> "必须与分红和拆分进行一定结合，才能同时收割二级市场"

## 七、互助盘判断与参与策略

### 7.1 识别互助盘

**关键特征**：
- 承诺固定收益/APY
- 收益来自新进资金
- 有清算/重启机制
- 出金有规则限制

### 7.2 风险评估

**关键指标**：
- 全局债务 vs 可清算资产
- 新入金速度 vs 出金速度
- 抽水比例是否透明
- 清算机制是否明确

### 7.3 参与策略

> "当我们研究一个盘子，最好的策略是二级市场套利，其下快速复制仿盘并发射，最下大资金冲入庞氏"

**具体操作**：
1. 在IDO抛压出清后，买入其矿币
2. 等他拉高以诱导入金
3. 当TVL与LP大小失衡，新入金减缓，复投资金与LP增量同时走低时，果断撤出

---

## 八、流动性范围与适用场景

### 8.1 互助盘的流动性特点

```
分红盘 < 拆分盘 < 互助盘
```

互助盘流动性范围**最高**，适合在流动性充裕的市场环境下运作。

### 8.2 适用时机

- ✅ 市场流动性充裕
- ✅ 用户追求高APY
- ✅ 需要快速扩大规模
- ❌ 熊市/流动性枯竭时期
