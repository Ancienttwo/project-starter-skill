# 分红盘详解（Dividend Ponzi）

> 来源：Open Rug 开源镰刀系列 - 三盘理论1
> 原文发布：2023年12月26日

---

## 一、定义与本质

### 1.1 基本定义

**分红盘**是指：一次性投入一整笔资金，随时间线性分红获得收益的资金盘模型。

### 1.2 特征矩阵

| 特征维度 | 属性 | 说明 |
|----------|------|------|
| 资金池 | ✅ 是 | 形成中心化资金池 |
| 自由进出 | ❌ 否 | 本金锁定，无法自由退出 |
| 固定收益 | ❌ 否 | 币本位分红，非金本位刚兑 |

### 1.3 本质特点

分红盘的核心特征：
- 入金形成**沉没成本**（矿机、质押等）
- 收益按**时间线性释放**
- 需要计算**回本周期**
- 用户风险偏好最**保守**（类似债权投资人）

---

## 二、庄家视角

### 2.1 盈利公式

```
profit(分红盘) = 总入金 - 总固定分红
```

### 2.2 崩盘逻辑

**崩盘条件**：新入金 < 固定收益利息

**了结方式**：停止资金池提取（关盘赚剩余本金）

### 2.3 操盘优势

传统分红盘项目方需要大量准备金刚兑收益，但Crypto分红盘（如矿币）可以采用**二级杠杆**：

> "1份拉盘可能产生5倍矿机销量"

买矿机相当于**锁死本金**，电费相当于随产量增加的**沉没成本**，无需考虑市场全局流动性是否足以让矿机原值卖出。

---

## 三、Crypto分红盘的革命性创新

### 3.1 比特币/POW对分红盘的革命

比特币是典型的分红盘：通过投入矿机与电费(入金)，获得固定回报(区块奖励)。

**三大跨时代革命**：

| 创新点 | 传统分红盘 | 比特币 |
|--------|------------|--------|
| 项目方 | 有明确项目方 | **没有项目方** |
| 收益本位 | 金本位刚兑 | **币本位，无刚兑** |
| 入金去向 | 被项目方回收用于提供收益 | **沉没成本，不被回收** |

### 3.2 从坐庄端看比特币

**优势**：
- ✅ 无需准备金
- ✅ 无实质拨比（区块奖励非金本位刚兑）

### 3.3 从用户端看比特币

"名义上"满足了所有"完美资金盘"的想象：

- ✅ 无入门费（虽然要算力和电费，但不是给项目方入金）
- ✅ 自由进出（没有锁仓，虽然有机会成本）
- ✅ 收益高（梦想多高就有多高）
- ✅ 无项目方（法律上没风险）

## 四、矿币盘的运作机制

### 4.1 矿币流动性来源

矿币原生没有流动性，需要去市场找流动性（有人愿意出价买出价卖），找他们的"披萨饼"（对比特币定价）。

这个过程：
- 要么如比特币**自然发生**
- 要么由人为催生，有人主动当**做市商**

### 4.2 矿币唯一注入本金的方式

> "唯一在矿币模型中注入本金的方式，就是部署算力，也就是所谓'买矿机'"

### 4.3 矿币第一个庄

比特币以后所有矿币的第一个庄，就是**矿机或者矿池**。

**正循环模型**：
```
矿工买矿机入金
    ↓
入金去拉盘
    ↓
价格上涨更多人买矿机/买币
    ↓
卖币/卖矿机获利
```

### 4.4 矿币持续条件

```
矿机销量 + 流动性 > 产出流通 → 可继续拉
```

### 4.5 庄家策略

矿商通过掌握算力掌握筹码结构，根据当下筹码状况：
- 尽可能拉盘抬价提高矿机销量
- 甚至可以"赚一级，亏二级"

> "记住，做市商更多是对当下筹码结构做出反馈，而非基于整个矿币模型基于生命周期做定价"

## 五、典型案例分析

### 5.1 Helium Mobile ($MOBILE)

**筹码分析**（截至2023年8月）：
- Helium矿机必须通过购买获得
- Miami主要基站由自身掌握
- 合理估计掌握50%以上Mobile产量
- 第一年50B产量为预挖
- 实际流通仅为30B左右
- 按当时价格0.00026计算，流通量仅价值78万U

**拉盘期望公式推演**：
- 假设到明年8月算力结构变化不大
- 按价格$0.0041，年新增抛压为$1200万
- 山猫矿机成本$450，平均日交易量$9M
- 销售增量预期需达到6700台以上

### 5.2 其他矿币案例

早期矿币（盘龄1-2年）代表：
- $KAS（Kaspa）
- $HONEY
- $SMH

> "Depin或者POST这些都只是fugazi，本质上都是矿币的别名"

### 5.3 Crypto Game = 矿游

**核心观点**：
> "几乎整个Crypto Game赛道，从Stepn、Axie Infinity到BigTime甚至LootRealms，本质上都是更为创新的矿机分红盘。"

**选择游戏赛道标的的关键**：
- ✅ 是否有清晰的矿业结构（矿机NFT - 产出逻辑）
- ✅ 是否有矿工群体（专业工作室）

> "crypto game需要的不是传统意义上的玩家，而是大量三班倒，18个人6个号，卖沙漏充水晶的专业工作室"

> "除了全链游戏选择走'正统性'(to Vitalik)之外，已经做出了价值回归的方向选择，那就是成为'矿游'"

## 六、分红盘的生命周期管理

### 6.1 分红盘的局限

分红盘绝非唯一解，甚至不是最佳解。

**问题场景**：
- 市场流动性匮乏
- 筹码结构变得更为分散
- 矿机盘进入第二阶段（如AR, FIL, XCH等）

**后果**：
> "没有后手的矿币，在失去上涨动能后，将不可避免的量价齐跌，失去流动性导致死局"

### 6.2 防崩盘调整策略

在筹码结构分散后进行调整防止崩盘：

| 策略 | 方法 | 效果 |
|------|------|------|
| 改变增量结构 | 降低算力或封号 | 减少新增供给 |
| 套娃新盘 | 新增分红盘或拆分盘 | 形成质押或消耗 |
| 找长期流动性 | 上所 | 获得外部流动性支撑 |

### 6.3 矿机分红盘完整剧情

```
卖矿机/扩矿工
    ↓
DEX或小所拉盘
    ↓
卖更多矿机
    ↓
上大所
    ↓
调整防崩盘
```

**重要提示**：
> "上DEX或小所可有效防止大所授信交易团队打对手盘"

## 七、信任成本分析

### 7.1 分红盘的信任成本最高

从盘型来说，信任成本从高到低依次为：
```
分红盘 > 拆分盘 > 互助盘
```

这主要由用户的**沉没成本**决定：沉没成本越大，信任成本越高。

### 7.2 典型信任建设方式

**矿币项目的信任成本投入**：
- 北美顶级机构排排坐（如KAS、HNT、SMH）
- 生态获奖、牌照背书
- 大咖站台

**根本原因**：
> "矿机是一次性不可逆投入，要算回本周期，信任成本高"

## 八、分红盘设计要点总结

### 8.1 设计清单

| 设计维度 | 关键考量 |
|----------|----------|
| 入金机制 | 设计沉没成本（矿机、质押、锁仓） |
| 收益释放 | 币本位优于金本位，避免刚兑压力 |
| 筹码控制 | 掌握算力/产出，控制筹码结构 |
| 流动性 | 从DEX→小所→大所逐步获取 |
| 防崩盘 | 预留套娃新盘或质押消耗机制 |
| 信任建设 | 根据入金规模匹配信用背书 |

### 8.2 核心公式

**盈利公式**：
```
profit = 总入金 - 总固定分红
```

**持续条件**：
```
矿机销量 + 流动性 > 产出流通
```

**崩盘触发**：
```
新入金 < 固定收益利息
```

## 九、与其他盘型的结合

### 9.1 分红盘在三盘结合中的角色

分红盘主要提供**锁定机制**：
- 锁定用户资金
- 减少抛压
- 形成质押需求

### 9.2 典型结合案例

**Restaking中的分红盘元素**：
- 下游"共享安全性"协议本质是分红盘
- 质押ETH获得分红
- 分红来自协议的安全租金
