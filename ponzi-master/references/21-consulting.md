## 二、项目方/操盘手引导

### 2.1 第一层：你想做什么？

```
Q: 你想做一个什么类型的项目？

A1: 矿机/质押/锁仓类 → 分红盘设计
A2: P2P匹配/借贷/互助类 → 互助盘设计
A3: 发币/NFT/铭文类 → 拆分盘设计
A4: 公链/生态系统 → 三盘结合设计
A5: 不确定 → 继续下一层问题
```

### 2.2 第二层：你的资源是什么？

```
Q: 你有什么资源优势？

A1: 有流量/社区资源 → 考虑分红盘或拆分盘
A2: 有技术/开发能力 → 考虑发币或DePIN
A3: 有资金/做市能力 → 考虑互助盘或做市商角色
A4: 有渠道/代理体系 → 考虑分红盘+代理模式
A5: 有叙事/PR能力 → 考虑拆分盘+叙事包装
```

### 2.3 第三层：你的目标是什么？

```
Q: 你的核心目标是？

A1: 快速套现 → 短期拆分盘/Meme
A2: 长期运营 → 分红盘+防崩设计
A3: 上交易所 → 分红盘→拆分盘路径
A4: 建生态 → 三盘结合设计
```

## 三、投资者/韭菜引导

### 3.1 第一层：你想分析什么？

```
Q: 你想了解什么？

A1: 某个具体项目 → 案例分析模板
A2: 某类项目的风险 → 盘型识别
A3: 当前市场该投什么 → 时机判断
A4: 如何判断项目好坏 → 学习三盘理论
```

### 3.2 项目分析决策树

```
Step 1: 这是什么盘？
├── 有矿机/质押/锁仓 → 分红盘 → 查01-分红盘详解
├── 有P2P匹配/借贷 → 互助盘 → 查02-互助盘详解
├── 主要靠发币/NFT → 拆分盘 → 查03-拆分盘详解
└── 多种特征组合 → 三盘结合 → 查06-可组合性

Step 2: 庄家怎么赚钱？
├── 卖矿机/收质押费 → 分红盘庄家利润
├── 吃点差/老鼠仓 → 互助盘庄家利润
└── 抛售代币/NFT → 拆分盘庄家利润

Step 3: 什么时候会崩？
├── 分红盘 → 筹码分散+上涨动能消失
├── 互助盘 → 全局债务>可清算资产
└── 拆分盘 → 新买盘<拆分需求

Step 4: 当前时机匹配吗？
→ 查04-受众论与时机论
```

### 3.3 红旗警告快速检查

| 警告信号 | 问题 | 风险级别 |
|----------|------|----------|
| 承诺固定金本位收益 | 互助盘高危信号 | 🔴极高 |
| 无明确清算机制 | 踩踏风险 | 🔴极高 |
| 出金规则模糊 | 可能跑路 | 🔴极高 |
| 拆分速度过快 | 即将崩盘 | 🟠高 |
| 庄家利润不透明 | 隐藏抽水 | 🟠高 |
| 与市场Beta不匹配 | 时机不对 | 🟡中 |
| 叙事过于复杂 | 认知成本高 | 🟡中 |

## 九、总结

### 9.1 引导原则

1. **先识别用户类型**：项目方/投资者/撸毛党/学习者
2. **再澄清具体需求**：做什么/分析什么/撸什么/学什么
3. **匹配相应方法论**：根据需求定位到具体文档
4. **提供操作框架**：给出可执行的分析步骤

### 9.2 核心引导流程

```
识别用户类型 → 澄清具体需求 → 匹配方法论 → 提供操作框架 → 确认理解
```

### 9.3 万能问题

当不确定用户需求时，问：

> "你是想**做项目**、**分析项目**、还是**学习理论**？"
