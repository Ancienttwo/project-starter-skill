# Mass Admission用户增长方法论（User Growth Strategy）

> 来源：Open Rug 开源镰刀系列 - 16
> 核心观点："Mass Adoption全是扯淡，Mass Admission才是真理"

---

## 一、Mass Adoption的谬误

### 1.1 常见归因谬误

> "每次市场不景气时，大家就会怪市场没吸引Web2用户，所以没有新资金进来"

**这个逻辑的问题**：
> "好比'非洲小孩会饿死，因为你在中国浪费饭菜'，合乎情绪但毫无逻辑"

### 1.2 Mass到底是谁？

> "不要笼统地说是传统Web2用户——Bullshit。'大众'是一群不同但可分类的个体"

**盲人开车的比喻**：
> "盲人不开国产电动车，不是因为他不支持国货，而是因为他没有驾照（没有需求）"

## 二、Crypto的真正受众

### 2.1 Crypto的边界

> "Crypto是有边界的。它所有特性都来自于其核心价值——无许可性，不是地球上大多数人用得上的"

**无许可性的应用场景**：
| 限制方 | 无许可性的价值 |
|--------|----------------|
| 银行 | 不让你自由开户自由付款，你偏要 |
| 政府 | 不让你投资炒作的不让赌的，你偏要 |
| 大公司 | 限制的言论，你偏要 |

### 2.2 "商人"特质

> "那些有需求的人绝不是多数人，大多数人是被体制规训的家畜。而是那些懂套利、反建制的'社会边缘人'——我把这种人的特性称为'商人'"

**商人的定义**：
- 未必真的经商，甚至未必有钱
- 注定是少数，否则统治无法维持
- "在商言商"
- 充分定价自己所有时间、精力、人脉等资源
- 把"自己"卖个好价钱
- 以赚钱为首要目标

### 2.3 市场饱和度分析

> "你觉得现在没有新用户，是因为市场即将达到自己的实际TAM极值"

**数据推算**：

| 指标 | 数据 |
|------|------|
| 全球成年人平均年均收入 | 23,000U |
| 假设10%进入Crypto | 18.4万亿潜在市场 |
| 币安2024年3月现货交易量 | 1.12千亿/月 → 13万亿+/年 |
| 剩余空间 | ~30% |

**更精确的估算**：
| 指标 | 数据 |
|------|------|
| 全球收入前10%人群 | 8亿人，占全球总收入52% |
| 2023年全球加密货币用户 | 5.2亿（Crypto.com数据） |
| 剩余空间 | ~30% |

**结论**：
> "受过中高等教育的人在2024年还没听过比特币才是稀奇"
>
> "但凡有可能被你adopt的，大概率已经接触过币了"

## 三、存量市场的正确理解

### 3.1 博彩行业类比

> "从行业整体角度用'Mass Adoption'的逻辑来说，博彩不存在什么增量市场存量市场——理论上全球的有钱人都是我的市场，因为有钱人都有赌性"

**Crypto与博彩的共同点**：
| 特点 | 说明 |
|------|------|
| 全球性 | Crypto甚至不受地区旅游业影响 |
| 增长逻辑 | 随全球通胀增长 |
| 业绩驱动 | 不是客流量，而是出码和下注频率 |

### 3.2 赌场的两种主力客户

| 类型 | 特点 | 贡献 |
|------|------|------|
| **高频用户** | 对一两种游戏特别精通且上瘾（老虎机、百家乐）| 天天去 |
| **VIP大户** | 高净值客户 | 占赌场80%收入 |

### 3.3 核心洞察

> "Crypto增长的本质就是培养高频和大户的过程"

## 四、Mass Admission范式

### 4.1 赌场的Admission逻辑

**赌场不存在adoption问题的原因**：
> "赌场提供了不可替代的需求（赌）的不可替代的解决方案"

**赌场对客户的明确态度**：
> "你来就是为了赢钱。进赌场你要赢钱，就要学赌场的规矩。是赌场放你进来赌(Admission)，而不是迎合你其他身份的'日常需求'(Adoption)"

### 4.2 荒谬的Adoption思维

> "你很难想像赌厅出来告诉你'我要做一个赌客的社交关系网络，让他们主动在我们的whisky bar里多消费而不是抱着赢钱的目的来这里'"

**博彩业的发展方向**：
- 从线下到线上
- 接受多种支付方式
- 更炫酷的老虎机主题
- 让UX更好

## 五、Crypto的Admission策略

### 5.1 核心定位

> "Crypto/Web3需要明确的是'让他们来赚钱!'"

**Admission的含义**：
```
放（Admit）世界上的"商人"们来这里试着赢钱
条件是学习Crypto的规矩
```

### 5.2 赢钱的方式

> "他们赢钱的方式，就是进行涉及金流的博弈（交易、质押、借贷等），一切围着这个逻辑走"

### 5.3 自然淘汰机制

| 被淘汰的 | 留下的 |
|----------|--------|
| 不按规矩玩的（如做项目不看三盘理论，只听KOL炒币） | 精通者 → 高频用户 |
| 玩得太菜导致破产的 | 佼佼者 → VIP厅 |

### 5.4 建设的真正含义

> "所谓建设，就是能让更多人来参加试炼，更快参加试炼，更多元酷炫的试炼，总下注更大，博弈更持续"

**建设方向**：

| 维度 | 具体内容 |
|------|----------|
| **降低门槛** | 让更多人来参加试炼 |
| **加快速度** | 更快参加试炼 |
| **丰富玩法** | 更多元酷炫的试炼 |
| **放大盘口** | 总下注更大 |
| **延长周期** | 博弈更持续 |
| **身份炫耀** | 赌厅卖表卖高奢，Crypto里是NFT？ |

## 六、用Admission逻辑看项目

### 6.1 为什么火的是这些？

> "用这个逻辑，去想一想，为什么火的是meme，火的是铭文，火的是Solana"

| 火的项目 | Admission逻辑 |
|----------|---------------|
| Meme | 低门槛、快速博弈、结果明确 |
| 铭文 | 新规则、新玩法、快速试炼 |
| Solana | 低成本、高速度、交易友好 |

### 6.2 为什么不火？

| 不火的项目 | 问题 |
|------------|------|
| ETH全系列(除Base) | 门槛高、速度慢、成本高 |
| VC币 | 规则不透明、筹码集中 |
| RWA | 需求模糊、不够刺激 |

### 6.3 最缺的是什么？

> "为什么我说最缺的是dev而不是'用户'？"

**Dev的作用**：创造新的"试炼场"，让用户有新的博弈机会

## 七、Admission设计清单

### 7.1 项目评估

| 检查项 | 问题 |
|--------|------|
| **赚钱导向** | 用户能明确知道怎么赚钱吗？ |
| **规则明确** | 博弈规则是否清晰？ |
| **门槛适中** | 进入门槛是否合理？ |
| **速度够快** | 结果反馈是否及时？ |

### 7.2 增长策略

| 错误做法 | 正确做法 |
|----------|----------|
| 迎合Web2用户日常需求 | 明确"来赚钱"的定位 |
| 降低赚钱门槛 | 降低参与试炼的门槛 |
| 做社交产品 | 做博弈产品 |
| 追求用户数量 | 培养高频用户和大户 |

### 7.3 产品设计

| 维度 | 设计原则 |
|------|----------|
| **入口** | 简单直接，一眼看懂 |
| **规则** | 学习成本可控，赌场规则 |
| **反馈** | 赢亏即时可见 |
| **进阶** | 精通者有更高收益可能 |
| **炫耀** | 提供身份标识 |

## 八、总结

### 8.1 核心认知转变

| 错误认知 | 正确认知 |
|----------|----------|
| 需要Mass Adoption | 市场已近饱和 |
| 要吸引Web2用户 | 要培养高频用户 |
| 要满足日常需求 | 要满足赚钱需求 |
| 缺用户 | 缺Dev和试炼场 |

### 8.2 增长公式

```
Crypto增长 = 高频用户培养 + 大户转化
         ≠ 新用户获取
```

### 8.3 建设公式

```
有效建设 = 降门槛 × 加速度 × 丰富玩法 × 放大盘口 × 延长周期
```

### 8.4 最后一句

> "Mass Admission是我看项目做项目的一个核心逻辑，甚至是第一个思考的逻辑"
