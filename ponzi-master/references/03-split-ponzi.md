## 一、定义与本质

### 1.1 基本定义

**拆分盘**是指：在系统总资金不变的条件下，倍增单位资金对应的资产数量，以吸引后续资金进场的盘子。

> "这里的'倍增'也就是拆分这个说法的来源"

### 1.2 特征矩阵

| 特征维度 | 属性 | 说明 |
|----------|------|------|
| 资金池 | ✅ 是 | 形成流动性底池 |
| 自由进出 | ✅ 是 | 可自由交易退出 |
| 固定收益 | ❌ 否 | 收益来自资产增值 |

### 1.3 核心逻辑

> "盘子的本质，是超速加杠杆，加杠杆→泡沫，泡沫→崩盘"
>
> "消泡沫最好的办法是提升实际产出；而最快的方法，是开另一个盘子"
>
> "当你能无限开盘并让盘中人无法自拔，你就掌握了开盘的无限宝石——拆分盘"

## 二、房产拆迁类比

### 2.1 经典话术解析

> "假如你有套50万的房，后来涨到了100万，此时政府需要征地，你房子要拆迁。国家补偿其他地段等值2套房各50万，总价还是100万。后来你的房子又升值到100万，又拆迁，两套房子就是200万，国家又补偿给你新城区4套房子各50万，总价还是200万。4套房子又涨到100万，就是400万"

### 2.2 隐含假设

用"流动性底池"视角看这个故事的两个隐含假设：

1. **需要后续不断有新人进来买房**，导致需要不断征地（一直有新的流动性进底池）
2. **你手上的房子始终没有卖出套现**（不断获得新空投却没有砸盘套底池）

### 2.3 传统拆分盘的承诺

古早拆分盘（如MBI）会承诺"电子股只涨不跌"，原理是**中心化关闭了降价套现的通道**。

也会说"不承诺年化收益，根据市场情况决定"，本意是让人产生托底心态，而不造成更多泡沫。

## 三、庄家视角

### 3.1 盈利公式

```
profit(拆分盘) = 项目方抛售总利润（市值利润了结）
```

### 3.2 市场平衡公式

```
新增购买 = 拆分后总价值 - 拆分后不卖的总价值
```

简化为：
```
新增购买 = 拆分后总房价 - 拆分后不卖的总房价
```

### 3.3 四大崩盘点

假设：
- 政府每次拆迁都是1套拆成2套
- 每N次拆迁房价都会倍增
- 拆迁前房产总价X
- 新移民买房买盘是K

**崩盘条件**：

| 序号 | 崩盘点 | 描述 |
|------|--------|------|
| 1 | 新买盘进不来 | 存量房主不抛售，但K仍小于需求 |
| 2 | 存量大幅抛售 | K值不变，存量房业主大量抛售令供给大于需求 |
| 3 | 拆分过快吸血 | K值不变，拆分次数N过大 |
| 4 | 拆分后价格起不来 | 常数2变小，价格在下一次拆分前无法倍增 |

## 四、真实案例：北印度房产危机

### 4.1 问题诊断

代入"亚洲国家北印度"的房产危机：

| 崩盘点 | 具体表现 |
|--------|----------|
| 新买盘进不来 | 北印度出生率暴跌，老龄化严重，城市化已超过6成 |
| 拆分过快 | 大基建和棚改货币化导致大量新增住房面积 |
| 存量抛售 | 疫情政策导致的主要城市治理信任危机 |
| 价格起不来 | 经济政策导致的市场信心不振 |

**结果**：北印度房价普跌甚至死亡螺旋

### 4.2 政策应对

由于北印金融系统房产作为抵押物比例过高，价格下跌将导致Luna式连环清算。

**短期政策选项**：

| 策略 | 对应崩盘点 |
|------|------------|
| 提高城镇化速率 | 提高刚需买盘 |
| 停止新增建筑审批 | 拆分降速 |
| 行政手段降低二手房流通 | 防存量抛售 |
| 舆论诱导 | 改变价格预期 |

## 五、Crypto拆分盘案例

### 5.1 FriendTech

**模型**：
- 用户接入Twitter账号铸造钥匙
- 钥匙通过AMM曲线定价，无锁定期
- 保持持仓可领积分

**拆分盘分析**：

| 维度 | 分析 |
|------|------|
| **拆分速度** | = 新具有影响力KOL涌入速度 |
| **拆分上限** | 各语种币圈KOL的总数 |
| **问题** | 马太效应明显，饱和度越高，拆分速度几何式下降 |

**新买盘/价格预期**：
- 理论上限是推特辐射币圈用户总数
- 但KOL上车快价格发现也快
- 拆分速度下降导致新用户进场热情降低

**存量抛售**：
- 盘子小、税点高，用户有低买高卖倾向
- 唯一阻止抛售的是积分空投预期 + Paradigm作保
- Post出现改政策后，保险接近报废

**崩盘原因**：
> "铭文崛起币市拉盘，Beta陡增。无法再拆分并作出赚钱效应的FT彻底熄火"

---

### 5.2 BNB与币安

**模型演进**：
- 最初：融资手段
- 19年前：手续费销毁只是销毁未解锁份额，无实质意义
- 19年：Launchpad/Launchpool开始出现持币权益
- 20年后：BSC作为gas和基础交易对

**拆分盘分析**：

| 维度 | 分析 |
|------|------|
| **拆分速度** | BNB拆分非均质 |
| | - Pancakeswap等DEX每天发无数币，但日交易量不足币安本体5% |
| | - CEX拆分主要来自Launchpad/pool频率 |
| **新买盘/价格预期** | 主要来自BNB刚需：BSC和Launchpad/pool收益预期与Beta关系 |
| **存量抛售阻止** | Launchpad/pool综合收益(平均ROI × 开盘频率) + BSC开盘收益率 |

**当前问题**：
> "高频率IEO价格不振导致BNB承压"
> "首发频率和单体涨幅本身就是一对矛盾"

---

### 5.3 以太坊

**革命性意义**：
> "以太坊可以说是拆分盘的'蒸汽机时刻'"
> "智能合约+gas打开了无限拆分的业界想象力"

**基本盘**：矿机分红盘（POS以后也是），通胀产出在ICO前只能成为卖压

**ICO时代**：开启以太的拆分盘时代

**拆分逻辑对比**：

| 平台 | 拆分逻辑 |
|------|----------|
| 币安首发 | 集全站注意力/流动性于新盘，系统级抽血 |
| ETH/BSC发币 | 需要项目方投入运营购买注意力（如空投），获取系统流动性 |

**以太拆分速度计算**：
```
拆分速度 = 年新高TVL / 流动性项目数
```

**ICO时代问题**：
- 存量抛售严重
- ICO项目方募资后砸盘
- 18年价格最低跌破100U

**Vitalik的应对**：
> "通过叙事以及以太内圈白手套，给拆分立规矩"

**叙事演进**：DeFi → NFT/SBT → LSD → Layer2 → Restaking

每个叙事的核心位置都非常有限，**控制了拆分的速率**

**反身性问题**：
- to V创业成主流
- 项目哲学愈发同质化
- 正统性=高估值，野路子/低流动性不再
- 拆分速率不足，Beta被BTC、Solana碾压

## 六、拆分盘的核心要求

### 6.1 三大平衡

> "拆分盘是资金盘的无限宝石，皇冠上的明珠，因为拆分盘是一门跨学科工程，是封闭系统推定最少的一种盘型"

**你需要**：

| 序号 | 平衡要求 |
|------|----------|
| 1 | 保证进场资金与拆分速率动态均衡 |
| 2 | 保证拆分收益率压住Beta |
| 3 | 保证存量抛压与进场资金均衡 |

### 6.2 关键变量

- **拆分速度**：新标的涌现速率
- **新买盘**：新进流动性
- **价格预期**：市场对增值的信心
- **存量抛售**：老持有者的套现意愿
- **市场Beta**：同类资产的基准收益率

---

## 七、拆分盘与流动性

### 7.1 流动性定价

> "套现需要流动性。AMM曲线告诉我们，每个价格都有对应的流动性，只是价格失准时流动性会低的可怕，乃至你手上的筹码根本卖不出去。"

### 7.2 隐含Beta

> "不管许诺与否，市场都会对你当前盘面情况进行一个市场心理定价"
> "这个定价往往是通过与其他项目对比得出，这里我们称之为隐含Beta"

### 7.3 最大敌人

> "一个拆分盘，最大的敌人是另一个收益更高更火爆的拆分盘"

---

## 八、流动性范围与适用场景

### 8.1 拆分盘的流动性特点

```
分红盘 < 拆分盘 < 互助盘
```

拆分盘流动性范围**居中**。

### 8.2 名义回报范围

```
分红盘 < 互助盘 ≤ 拆分盘
```

拆分盘名义回报范围**最高**（通过资产增值实现高倍数）。

### 8.3 适用时机

- ✅ 有持续的新标的可拆分
- ✅ 市场Beta预期合理
- ✅ 存量持有者有锁仓动力（如积分、权益）
- ❌ 拆分饱和、无新标的
- ❌ Beta被其他生态碾压

## 九、拆分盘设计要点

### 9.1 设计清单

| 设计维度 | 关键考量 |
|----------|----------|
| 拆分机制 | 新标的如何产生（新KOL、新项目、新NFT） |
| 拆分速率 | 如何控制（叙事、审核、门槛） |
| 锁定机制 | 如何阻止存量抛售（积分、权益、预期） |
| 流动性 | 底池设计、AMM曲线选择 |
| Beta管理 | 如何保持收益率竞争力 |

### 9.2 核心公式

**盈利公式**：
```
profit = 项目方抛售总利润
```

**市场平衡**：
```
新增购买 = 拆分后总价值 - 拆分后不卖的总价值
```

**崩盘条件**（任一触发）：
1. 新买盘 < 需求
2. 存量大幅抛售
3. 拆分过快
4. 价格无法倍增

## 十、与其他盘型的结合

### 10.1 拆分盘在三盘结合中的角色

拆分盘主要提供**增值预期和退出流动性**：
- 创造新的投资标的
- 提供价格上涨预期
- 通过AMM提供退出通道

### 10.2 典型结合案例

**Restaking中的拆分盘元素**：
- 不断拆分出新的"共享安全性"分红盘
- 每个新协议都是一个新拆分标的
- 用新标的吸引增量资金

**以太坊生态**：
- 基本盘是分红盘（POS质押）
- 通过LSD释放ETH筹码
- 拉拆分盘（新项目生态）
