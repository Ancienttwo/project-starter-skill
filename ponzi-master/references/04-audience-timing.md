## 一、受众论基础

### 1.1 核心观点

> "三盘庞氏就是对受众需求的一个'结果'，你可以用三盘对已知条件做分析设计，但无法用它来'春江水暖鸭先知'。但受众论可以。"

> "受众决定需求，需求导致供应。"

### 1.2 风险偏好与盘型匹配

从受众角度，分红、互助到拆分是**风险偏好不断上升**的过程：

| 盘型 | 风险偏好 | 传统金融类比 | 用户特征 |
|------|----------|--------------|----------|
| 分红盘 | 低 | 债权投资人 | 追求稳定回报，接受锁定 |
| 互助盘 | 中 | 高利贷者 | 追求高APY，接受规则约束 |
| 拆分盘 | 高 | 股权投资人 | 追求高倍数，接受波动风险 |

### 1.3 信任成本与盘型

从盘型来说，信任成本从高到低依次为：

```
分红盘 > 拆分盘 > 互助盘
```

**原因**：由用户的**沉没成本**决定，沉没成本越大，信任成本越高

**实践含义**：
- 矿币项目需要大背书（北美顶级机构）
- 买新L1生态币需要看公链背书
- DeFi二池和土狗只需看合约

## 二、世代变迁

### 2.1 旧世代特征（60后-90前）

三盘理论和开源镰刀大部分实践基于2013-2021年网络传销时代，受众主要是60后-90前。

**群体特征**：
- 接受中国式家畜教育规训
- 赶上中国经济上升期
- 盲目自信，爱装逼
- 迷信权威
- 思考能力有限

**有效话术**：
- "风口"
- "慢慢变富"
- "长期主义"

> "只要你找到了'高大上'的叙事，和角度正确的PUA，还是相对容易让他们买单的"

### 2.2 新世代特征（PVP世代/05后）

**成长背景**：
- 获取信息更快
- 大多撞上中国经济衰退期
- 对现有体制有幻灭感和反叛倾向
- 从未经历任何红利时代
- 本地竞争空前汹涌

> "非零和博弈对于这个世代来说，就好像大哥大功能机一样遥远（对他们来说智能手机与生俱来）"

**核心特点**：
- 习惯于零和博弈环境
- 对PVP概念没有道德评判或厌恶
- 追求"别人钱进自己兜"的速度
- 注意力短暂，漠视不够刺激的事物

### 2.3 世代更替的影响

> "这几天表达对Pump直播厌恶的，绝大多数都不是PVP世代年龄段，因为只有足够猎奇足够刺激才能获得他们的关注和参与"

**可预见的趋势**：
- 世代更替必然摧毁"普世价值"
- 未来的Pump风格直播或许会成为平常
- 零和在本世纪中叶前除罪化

## 三、直销与层级制度的衰亡

### 3.1 全球直销市场萎缩

- 2022年开始全球直销市场正式萎缩
- 2023年中、韩、日三个直销大国快速倒退
- 传统基于团队层级销售机制的分润机制正在衰亡

### 3.2 对盘圈的影响

**流量来源变化**：
- 多数盘圈团队来自直销
- 直销人口蓄水池下跌，可发展团队更少
- 团队之于盘子 = 流量、入金

**新模式取代旧模式**：
- 直播和小红书带货取代团队层级体系
- 没有团队一样可以有流量有入金
- 更快见效

### 3.3 新一代CX模式

> "年轻一代根本不买60-90一代的账。他们更会做流量，CX更赤裸。"

**特征**：
- 以冲土狗社区或KOL带队形式
- 层级返佣逻辑逐步被淘汰
- 追求更快见效来做大流量和影响力

## 四、去中心化共谋

### 4.1 阴谋集团的形成

低流动性市场（数藏、NFT、铭文、Memecoin）绝非只因一个具体的"庄"而成就，而是**利益群体合力**的结果：

- 项目方
- 做市商
- KOL
- 早期发现者/传播者
- 早期交易者

> "他们都是阴谋集团的一部分，尽管未必相识。我管这个叫去中心化共谋"

### 4.2 关键问题

比起分析市场机制，更重要的问题需要回答：

1. 什么样的人会有动力参与这样的市场？
2. 他们是怎样知道这样的市场存在，并且在接触到后立刻all in其中？
3. 他们为什么会主动CX？
4. 他们如何在这个市场建立优势，又是怎么明确知道自己具有优势？

### 4.3 邪教问题

> "邪教真的是跟着项目走的吗？还是说邪教是人的邪教，'邪教'项目只是这个邪教社团的一个阶段性仪式？教主到底是SPX6900，还是Murad本身？"

## 五、时机论（Timing Theory）

### 5.1 核心观点

> "神盘之所以火起来，未必是本身多优秀，更可能是因为他们起盘的时机。"

> "没有不能开盘子的时间，只有不适合在具体某个时间开的盘子"

**常见误区**：
> "牛市才适合开盘子这种说法是错的，甚至很多时候恰恰相反"

**例证**：2021年牛市，盘圈年化300%与每天暴涨的二级相比索然无味，地推盘难起来

### 5.2 两个核心预期

决定具体时间段某类盘子适不适合开的主要因素：

#### 5.2.1 相对市场的流动性预期

**定义**：因为对当前市场实际流动性的判断而造成的预期

**流动性的意义**：
> "你能够把账面上本金和收益从盘子里套出来的可能性"

**影响**：
- 低流动性：跑得不够快就套不出钱
- 高流动性：可能很难拉盘起倍数

**观察指标**：
- 具体链上DEX的交易量
- NFT的交易量
- 等具体数据

#### 5.2.2 市场Beta预期

**定义**：同类盘子的名义收益率或涨幅导致的预期

**包括**：
- Lending/LSD等生息方式的互助和分红盘收益率
- 币安上新、最新土狗/NFT等拆分盘的名义涨幅

**作用**：
> "市场Beta决定了你的盘子对于散户的吸引力"

**用户行为**：
> "绝大多数参与者ape进你的盘子时首先会去对比评判可能能获得多少收益/涨幅，再回过头看流动性来评估风险（甚至很多人不看，这是区分新旧韭菜的一个标准）"

## 六、盘型与时机匹配

### 6.1 流动性范围排序

每个盘子类型都有自己的流动性范围，从低到高：

```
分红盘 < 拆分盘 < 互助盘
```

### 6.2 名义回报范围排序

不考虑动态拉人头，从名义回报范围（名义利率/涨幅）：

```
分红盘 < 互助盘 ≤ 拆分盘
```

### 6.3 交易机制流动性排序

```
挂单交易(NFT/铭文) < AMM < 订单簿(CEX)
```

**解释**：
- 挂单交易：不需要准备流动性
- AMM：任何流动性上都能报价成交，仅需很小底池
- 订单簿：需要订单匹配和中心化做市商，需准备不低于交易所要求的流动性铺单

### 6.4 时机判断原则

**核心规则**：
- 盘子自身流动性范围 < 市场流动性预期 → **不适合**
- 盘子名义回报范围 < 市场Beta预期 → **不适合**
- 反之亦然

## 七、时机判断实例

### 7.1 低流动性市场案例

**场景**：2023年9月
- ETH NFT交易量前十每天才30万美元
- Dextool前十土狗交易量才100万多

**适合**：
- NFT、铭文等挂单交易形式
- 非常低的流动性可以向上挂单拉几十倍吸引眼球
- 不用准备扫地板或前置流动性成本

**不适合**：
- 直接上CEX发币的分红盘
- 上抹茶都需要保证多少流动性

### 7.2 高Beta市场案例

**场景**：牛市中
- 随便一个币每天10%涨幅
- 年化200%的分红盘不能随进随出

**结论**：分红慢盘没有吸引力

**场景**：熊市中
- 美联储利息4.5%
- 链上DeFi一般只能做到4%甚至更低

**结论**：年化100%的分红盘会有人跟

### 7.3 历史对号入座

| 现象 | 解释 |
|------|------|
| MoneyArk快速死亡 | 分红盘在高Beta市场没吸引力 |
| 一月份铭文熄火 | 流动性/Beta环境变化 |
| 404昙花一现 | 低流动性标的在流动性增加后失去优势 |
| Meme持续火爆 | 持续匹配市场Beta预期 |
| 符文表现差 | 挂单交易在高流动性市场失去优势 |
| VC盘拉跨 | 大估值上BN OK vs 市场Beta不匹配 |

## 八、舆情监测方法

### 8.1 三种监测方式

#### 方法1：散户舆情监测

**操作**：找到低流动性标的，看散户是夸倍数的多还是骂卖不出去的多

**解读**：
- 夸倍数多 → 流动性预期好
- 骂卖不出去多 → 流动性预期差

#### 方法2：VC bro舆情监测

**操作**：观察VC bro的内容输出

**解读**：
- 不断输出各种叙事相关内容 → 相关市场流动性好
- 开始质疑某些叙事 → 市场流动性减弱

#### 方法3：博主流量监测

**操作**：观察不同类型博主的流量

**解读**：
- 土狗博主发推阅读量互动数大 → 市场流动性好
- 输出类型博主/"深度分析"博主流量重回10K以上 → 流动性差

## 九、面向新世代的策略

### 9.1 受众选择：精准适配

**选项A：老用户（00年前出生）**
- 能够接受长周期、复杂逻辑和大资金交易
- 适配DeFi或包装成新叙事的DeFi产品
- 借鉴交易所大客户策略
- 上个周期的方法论依然有效

**选项B：PVP世代**
- 需要完全不同的设计思路

### 9.2 设计策略：庞氏与博彩结合

**高维度设计**：三盘理论依然有效

**Pump Fun案例分析**：
> "本质是一个互助拆分盘，通过高速拆分标的，反复的让盈利在庄闲间换手——链上的DEV们在其他币种里一样是韭菜，生生不息。"

**To C设计原则**（面向PVP世代）：
- 流程极短，盈亏结果快速显现
- 夸张的盈亏感官刺激
- 拒绝复杂的回本周期与利润率叙事
- 呈现出重博彩元素的产品

### 9.3 执行策略：快闪

**制造第一印象**：仅有30分钟获得用户关注

**关键策略**：
- **发币即营销**：通过发币制造流量和关注
- **先有meme，后有项目**：以meme和猎奇文化吸引眼球，项目逻辑反而次要
- **简化流程**：将复杂机制隐藏于后端，第一阶段只突出"快进快出"的体验

> "对PVP用户，奇观化的刺激甚至胜过讲解机制。Dev冲过去对着他们露鸟获得他们追随的可能性都比开一节space来的大"
