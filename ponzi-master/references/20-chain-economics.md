# 公链经济学方法论（L1 Economics）

> 来源：Open Rug 开源镰刀系列 - 19（ETH案例）
> 核心观点："ETH逃不出三盘：今天的雷在2018年就已经埋下"

---

## 一、公链的三盘属性

### 1.1 公链是经典分红盘

> "以太坊本身是一个经典分红盘，不论是在POW时代还是POS时代，主要抛压来自新的产出"

### 1.2 需求侧双因素

| 因素 | 说明 |
|------|------|
| **原生因素** | 因公链技术发展，产生大量以本币计价的拆分盘，推动对本币的需求 |
| **外部因素** | 生态外需求和宏观环境 |

---

## 二、ETH需求侧分析

### 2.1 原生因素：历史案例

| 时期 | 主叙事 | 效果 |
|------|--------|------|
| 2017年 | ICO | 大量项目以ETH计价 |
| 2020/21年 | DeFi | YFI、CRV、COMP等链上ETH本位定价 |

### 2.2 本轮的问题

**主叙事**：L2和Restaking

**问题分析**：

| 叙事 | 问题 |
|------|------|
| **L2** | 生态项目与主链高度重合，无法引起爆炸式交投繁荣 |
| **PointFi/Restaking** | 本质是锁定ETH降低流动，而非让更多资产以ETH定价 |

**关键洞察**：
> "大型restaking项目的定价权如Eigen、Rez、Ethfi都在交易所(USDT本位)，而不像上一轮YFI、CRV、COMP一样在链上(ETH本位)"

### 2.3 核心结论

> "只要没有大量新资产用ETH本位计价，用户就没有必要持有ETH"

### 2.4 燃烧机制的削弱

**EIP1559的设计**：ETH主要功能是结算层，大型DeFi的清结算都发生在主链上

**L2的影响**：
> "L2跟主链功能高度重合，导致大量该类需求都被分流去了L2，而这类交易导致的燃烧却只有原先的零头，削弱了对ETH的需求"

### 2.5 外部因素

| 上轮 | 本轮 |
|------|------|
| 宽松周期 | 紧缩周期 |
| 灰度信托（貔貅，只进不出） | ETF（可进可出） |

**ETF数据**：
> "开通ETF已经一个月，总净流出-140.83K，绝大多数通过灰度。相当于ETH新旧鲸鱼都在通过ETF套现"

## 三、供给侧：POW vs POS

### 3.1 POW时代的成本结构

**矿工获取ETH的成本**：

| 成本类型 | 构成 |
|----------|------|
| **固定成本** | ETH矿机成本（一次性不可退回） |
| **增量成本** | 电费、托管成本、意外成本 |

**关键特点**：
- 成本是**法币本位**
- 成本是**不可退回的沉没成本**
- 矿机寿命有限，总产出固定

**成本公式**：
```
每个ETH获取成本 = 法币总成本 / 总ETH产出
```

### 3.2 POW的价格支撑机制

> "当以太坊法币计价的市价低于获取成本（关机价），矿工就不会卖出，因为会亏本"

**成本抬升效应**：
- 矿机推陈出新，每一代更贵
- 每一轮市场挖矿竞争更激烈
- 产出降低，难度增加
- 电费和托管费水涨船高
- 政府监管压力随行业扩大变大

**结论**：
> "总增量成本变大，变相抬高了ETH地板价"

### 3.3 POS时代的成本结构

**验证者获取ETH的成本**：

| 角色 | 成本 |
|------|------|
| **验证者** | 提供基础设施费用（人员、服务器） |
| **质押者** | 质押ETH的机会成本 + 手续费 |

### 3.4 POS的关键问题

**区别分析**：

| POW | POS |
|-----|-----|
| 法币本位成本 | 几乎无法币成本 |
| 矿机报废限制产出 | 理论可承载无限质押 |
| 存在"关机价" | 不存在"关机价" |
| 矿工维持价格下限 | 可无限挖卖提 |

**核心问题**：
> "质押者不会像矿工那样维持ETH价格的下限，而是可以无限挖卖提"

> "即使质押ETH入场均价是上轮平均价格，这样的机制也不可能不断抬高ETH地板价。只要新增以太坊数量为正，价格就不断会承压"

## 四、ETH的历史教训

### 4.1 2018年的雷

> "这是一个悲伤的故事"

**ICO时代的问题**：
- 大量ETH计价ICO项目无序砸盘
- 最低砸到$100以下
- 拆分速率极高，但没有DEX用ETH本位计价交易套现
- 项目方只能砸盘ICO代币和ETH换USDT

**戴维斯双杀**：
> "ICO Beta收益骤减，机会成本高于持币，形成戴维斯双杀"

### 4.2 应对策略的形成

**Vitalik和基金会的选择**：
> "从18年ICO的经历太过惨痛，不断强调路线图、主叙事、正统性，形成'核心圈'开发者、VC"

**目的**：
> "让筹码集中到Eth Aligned的一致行动人手中而非每个人，从而防止无序拆分和无序抛压"

### 4.3 策略的副作用

> "DeFi Summer的成功更加强了这种体制固化"

**最终演变**：
| 现象 | 结果 |
|------|------|
| "to V创业" | 拆分速率过低 |
| "清真 = 高估值" | 市场Beta跑不过竞对 |
| 能承接到流动性的dev和盘子骤降 | 生态活力下降 |

### 4.4 多重因素叠加

```
L2削弱燃烧 + POS无成本抛压 = 抵消所有防抛压努力
```

> "最终酿成了今天的悲剧"

## 五、公链设计的教训

### 5.1 分红盘设计原则

> "分红盘想要长治久安，就不要拍脑袋创新"

**核心要求**：
1. 形成**金本位计价**的固定成本和增量成本
2. 随资产流动性的提高，不断提高成本线
3. 抬高资产价格下限

**参考标准**：
> "如果实在不会，再回去看看BTC成本模型"

### 5.2 拆分盘设计原则

> "做拆分盘降抛压只是缓兵之计"

**真正目的**：
1. 让母币变成**计价资产**
2. 让持有不依赖于母币本身的涨幅
3. 扩大需求端和流动性

## 六、公链经济学检查清单

### 6.1 需求侧评估

| 检查项 | 问题 |
|--------|------|
| **本币计价资产** | 有多少资产以本币计价？ |
| **链上定价权** | 定价权在链上还是CEX？ |
| **燃烧机制** | 燃烧量是否足够？ |
| **外部需求** | 有没有灰度/ETF类需求？ |

### 6.2 供给侧评估

| 检查项 | 问题 |
|--------|------|
| **成本结构** | 是法币本位还是币本位成本？ |
| **价格下限** | 有没有"关机价"机制？ |
| **抛压来源** | 新增产出的抛压有多大？ |
| **成本递增** | 获取成本是否随时间递增？ |

### 6.3 生态评估

| 检查项 | 问题 |
|--------|------|
| **拆分速率** | 新项目发行速度如何？ |
| **Beta收益** | 生态项目Beta是否跑赢竞对？ |
| **开发者活跃度** | 有多少活跃dev？ |
| **筹码分布** | 是否过度集中？ |

## 七、POW vs POS对比总结

### 7.1 成本结构对比

| 维度 | POW | POS |
|------|-----|-----|
| **成本本位** | 法币 | 币本位 |
| **固定成本** | 矿机（不可退回） | 几乎没有 |
| **增量成本** | 电费+托管（持续法币支出） | 机会成本（币本位） |
| **成本趋势** | 递增 | 稳定或递减 |

### 7.2 价格影响对比

| 维度 | POW | POS |
|------|-----|-----|
| **价格下限** | 有（关机价） | 无 |
| **抛压控制** | 自然限制 | 无限制 |
| **长期趋势** | 成本抬升价格 | 持续承压 |

### 7.3 设计建议

| 目标 | POW方案 | POS方案 |
|------|---------|---------|
| **创造价格下限** | 天然具备 | 需要其他机制补充 |
| **控制抛压** | 矿工惜售 | 需要锁仓/燃烧机制 |
| **长期价值** | 成本递增支撑 | 需要需求侧驱动 |

## 八、总结

### 8.1 核心认知

> "公链逃不出三盘理论"

### 8.2 ETH教训

| 教训 | 说明 |
|------|------|
| **POS削弱价格下限** | 无关机价机制 |
| **L2分流燃烧** | 削弱需求侧 |
| **正统性过强** | 降低拆分速率 |
| **定价权外移** | 大项目在CEX定价 |

### 8.3 公链经济学公式

```
公链健康度 = (本币计价资产量 × 链上定价权) / (产出抛压 × 成本结构弱势)
```

### 8.4 设计原则

1. **POW优于POS**（从价格支撑角度）
2. **本币计价**优于稳定币计价
3. **链上定价权**优于CEX定价权
4. **递增成本结构**优于固定成本结构

### 8.5 最后一句

> "BTC成本模型仍是分红盘的标杆"
