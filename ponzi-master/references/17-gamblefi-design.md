# 博彩逻辑设计方法论（GambleFi Design）

> 来源：Open Rug 开源镰刀系列 - 17、23
> 核心观点："Crypto主线只围绕三样东西：邪教、赌场、资金盘"

---

## 一、博彩的第四次工业革命

### 1.1 博彩行业的三次工业革命

| 时代 | 革命 | 变化 |
|------|------|------|
| 19世纪末 | 第一次：拉杆老虎机 | 进入工业时代 |
| 1960年代 | 第二次：老虎机电子化 | 奖池积累成为可能 |
| 1990年代 | 第三次：线上博彩 | 无国界化 |

**共同点**：改变了博彩的参与方式（自动化→多媒体化→无国界化）

**未改变的**：博彩游戏的本质规则（百家乐、blackjack、老虎机基本规则）

### 1.2 博彩行业的困境

> "GenZ时代的人群对传统博彩的参与率越来越低"

**根本原因**：与游戏行业的竞争

**年轻人的态度**：
- 不希望接受老一代定下的规则
- 对规则没有话语权
- 不具有平等的监督权力（包网可以卡提现ban IP，赌场随时改杀率）

### 1.3 第四次革命：Crypto

> "年轻人的核心需求仍然是赚钱，但他们希望自己能够掌控游戏"

**Crypto能实现**：
- 智能合约进行强制性结算
- 智能合约可以成为赌具本身

**核心共识**：
> "Crypto已经是博彩行业正在发生的第四次工业革命"
>
> "博彩也是已知的最靠谱的Crypto项目产生真实收益的方式"

## 二、博彩的三个核心逻辑

### 2.1 博彩游戏三要素

| 要素 | 问题 |
|------|------|
| **投注** | 以什么形式投注？ |
| **判定胜负** | 怎么判定胜负？ |
| **决定胜负** | 谁决定胜负？ |

### 2.2 映射到Crypto交易

| 博彩要素 | Crypto交易 |
|----------|------------|
| 投注 | 发行 |
| 判定胜负 | 交易 |
| 决定胜负 | 做市 |

> "从Uniswap到NFT、铭文、数藏、FT、Pump等，抛开庞氏部分，单从交易来说，只是对这三个象限进行修改后的排列组合"

## 三、案例分析：Pump = Crash游戏

### 3.1 Crash游戏规则

博彩平台（Stake、Bitcasino、Rollbit）都有的经典游戏：

```
1. 选择投注金额
2. 选择持有倍率（愿意持有到多少倍兑现）
3. 买定离手
4. 系统随机给一个崩溃点，从1x开始向上走直到崩溃

结果：
- 持有倍率 > 崩溃点 → 输钱
- 持有倍率 < 崩溃点 → 赢（拿走利润扣手续费）
```

**举例**：
- 下注100块
- 1%几率开盘1x就崩，回报率99%
- 设定持有1.98x倍率
- 胜率理论上 = 99/1.98 = 50%

### 3.2 Pump的Crash本质

| Crash游戏 | Pump对应 |
|-----------|----------|
| 选择投注金额 | 内盘塞多少 |
| 倍率判定 | Bonding Curve或Raydium AMM曲线 |
| 谁决定胜负 | dev或其他大户（拉盘=高倍数，砸盘=亏损归零） |

> "Pump的机制其实就是赌博游戏Crash，只是套了meme币的壳子"
>
> "如果你觉得Pump过度PVP，其实也没错，因为Pump本身就是二元博彩的内核"

## 四、博彩货币化 vs 加密货币博彩

### 4.1 传统包网模式的问题

**Rollbit/Shuffle的本质**：还是包网

**包网的竞争**：早就是红海
- 唯一竞争点：运营（体验金、包赔、返佣）
- 核心逻辑：做对手盘吃客损
- 运营公式：流量 × 转化比 × 客单价 = GGR

**骨干团队特点**：
> "前大厂被优化员工，游戏/互联网流量逻辑重度路径依赖者，无法理解crypto时代的逻辑"

### 4.2 新一代需求

> "GenZ赌性比前辈只强不弱，但他们要的是自己能够定义赌局，能够有自己的'门道'或Alpha"

### 4.3 Crypto资产逻辑的优势

Token/NFT/铭文等资产逻辑让：
- 简单的"赌"包装成"交易"
- 名义上或实质上添加更多博弈选择
- 通过桥接外部流动性放大盘口体量
- 降低可感知博弈风险

> "这其实就是Pump们的革命性——货币化赌局，赌具化货币"

## 五、庞氏逻辑 vs 赌场逻辑

### 5.1 必须明确划分

> "Ponzi和赌场是可以交叉的，但作为设计者，你要明确自己的受众是什么人"

### 5.2 受众差异

| 类型 | 受众特点 | 不匹配的设计 |
|------|----------|--------------|
| **资金盘** | 很多人因风险厌恶才玩盘子（如分红盘玩家） | 给他们赌盘没用 |
| **赌盘** | 追求刺激和Alpha | 复杂的分红机制 |

### 5.3 生态选择

| 类型 | 应该找的生态 |
|------|--------------|
| **赌盘做生态** | 开小赌盘（土狗）的 |
| **拆分盘做生态** | 受众目的与母盘操盘方向一致的 |

> "一定要首先明确自己是赌盘还是资金盘，然后自己操盘的目的是什么，以终为始"

## 六、PVP世代的设计要求

### 6.1 PVP世代特征

| 特征 | 说明 |
|------|------|
| **零和适应** | 天然适应crypto，对PVP无道德评判 |
| **速度追求** | 追求加快"别人钱进自己兜"的速度 |
| **注意力短暂** | 漠视不够刺激的事物 |
| **极端化偏好** | 需要极端化的点子、项目和视觉表达 |

### 6.2 To C设计原则

| 原则 | 说明 |
|------|------|
| **流程极短** | 盈亏结果快速显现 |
| **夸张刺激** | 夸张的盈亏感官刺激 |
| **拒绝复杂** | 拒绝复杂的回本周期与利润率叙事 |
| **重博彩元素** | 呈现出来的东西就是重博彩元素的产品 |

### 6.3 执行策略：快闪

| 策略 | 说明 |
|------|------|
| **30分钟法则** | 仅有30分钟获得用户关注 |
| **发币即营销** | 通过发币制造流量和关注 |
| **先meme后项目** | 以meme和猎奇文化吸引眼球，项目逻辑次要 |
| **简化流程** | 将复杂机制隐藏于后端，只突出"快进快出" |

> "对PVP用户，奇观化的刺激甚至胜过讲解机制。Dev冲过去对着他们露鸟获得他们追随的可能性都比开一节space来的大"

## 七、博彩设计检查清单

### 7.1 三要素设计

| 检查项 | 问题 |
|--------|------|
| 投注形式 | 用户如何投入？（买币/mint/质押） |
| 判定机制 | 盈亏如何判定？（价格/曲线/时间） |
| 决定者 | 谁控制结果？（dev/做市商/随机） |

### 7.2 受众判断

| 检查项 | 问题 |
|--------|------|
| 受众类型 | 我的受众是资金盘玩家还是赌狗？ |
| 风险偏好 | 他们的风险偏好是什么？ |
| 刺激阈值 | 需要多大的刺激才能吸引他们？ |

### 7.3 PVP适配

| 检查项 | 问题 |
|--------|------|
| 反馈速度 | 盈亏结果多快显现？ |
| 视觉刺激 | 视觉呈现够极端吗？ |
| 流程简化 | 用户30分钟内能理解吗？ |

## 八、总结

### 8.1 核心认知

> "博彩的货币化是未来，而不是博彩+加密货币"

### 8.2 设计公式

```
博彩逻辑Crypto项目 = 投注形式 × 判定机制 × 决定者
                   = 发行 × 交易 × 做市
```

### 8.3 两条路径

| 路径 | 适合 | 设计重点 |
|------|------|----------|
| **庞氏路径** | 风险厌恶者 | 沉没成本、年化收益 |
| **博彩路径** | PVP世代 | 速度、刺激、简化 |

### 8.4 最后一句

> "祝各位老板开盘得利，我们西港顶峰相见！"
