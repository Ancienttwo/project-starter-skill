# 空投设计方法论（Airdrop Design Methodology）

> 来源：Open Rug 开源镰刀系列 - 13、15、16
> 核心观点："空投是变体资金盘，本质是市场营销成本"

---

## 一、空投的本质理解

### 1.1 为什么要空投？

项目方的根本利益是**卖币利益最大化**。空投相当于白送，如果能把每一枚币都卖出去，为什么要白送？

**原因只有三个**：

| 原因 | 说明 |
|------|------|
| **传播度不支持** | 不做空投直接ICO很难传播，KOL找不到无风险抓手宣传 |
| **合规不支持** | 直接ICO/IEO在很多国家不合规，上不了大所 |
| **条款不支持** | 交易所严格监督代币解锁，无法在上币伊始套现 |

### 1.2 空投三型模型

根据诉求不同，空投设计分为三型：

| 类型 | 目标 | 受众 | 核心公式 |
|------|------|------|----------|
| **型一** | 做数据给VC和交易所看 | VC/交易所 | 收益 = VC投资+上币流动性 - 实际空投成本 |
| **型二** | 给IDO和上币做流量 | 一级二级散户 | 撸毛党和赌狗通过"赚钱预期"合二为一 |
| **型三** | 从空投过程中牟利 | 羊毛党本身 | 把撸毛党作为营收来源的资金盘设计 |

## 二、空投阶段设计

### 2.1 第一阶段：搞定领投前

**特征**：没有足够背书，用户信任成本高

**策略**：
- ✅ 0成本或低成本活动
- ✅ 简单但重复的任务（提高时间沉没成本）
- ✅ 不惜一切做数据，**无需考虑后续兑现**
- ✅ 不限制多账号，让脚本制作简单

**数据类型选择**（四选一）：
```
TVL | 交易量 | DAU | 协议收入
```

**优先选择**：测试网领水交易（刷交易量）、注册点点点（刷DAU）

**关键动作**：
1. 将任务进程统一到可视化标准（如积分、LXP）
2. 找英语系大V发测试网教程
3. 积分发行不需要合理，与最终空投无关

> **案例**：Blast上来就让存钱给利息，是因为铁顺有Blur背书，信任成本不是问题，熊市直接上TVL成本最低

### 2.2 第二阶段：搞定交易所前

**交易所三要素**：合规、新用户、新买盘

**核心策略**：
```
做拆分盘，让用户尽可能多花钱，抬高他们未来的筹码成本
```

**三个必做动作**：

| 动作 | 目的 |
|------|------|
| **增加活动维度** | 覆盖更多资产标的和交易行为，增加后续操作空间 |
| **对外亲疏不辨** | 不让外界知道谁是亲儿子，保持相对平均，浑水摸鱼 |
| **对内明确阵营** | 真正的亲儿子是能吃到客损的项目，后续给予倾斜 |

> **案例**：ZKSYNC即使原生DeFi基建也很难分辨与Matter Labs的关系，最终混进去不少归零土狗拿到空投

### 2.3 第三阶段：TGE前（算账时刻）

**核心公式**：
```
首日交易所总买盘 ≥ 预计首日均价 × (TGE解锁 + TGE实发空投)
```

**倒推逻辑**：

```
举例：
- 当前估值：8000万
- 预计TGE估值：2亿
- VC无解锁，三个二线所买盘预计300万
- 结论：首日实发空投上限 ≤ 1.5%
```

**Blast案例验算**：
```
- 初始释放：17%（170亿）
- 主力所：Upbit/Bithumb/Bybit
- 预估交易量：3-5亿美元
- 纯买盘（2成换手）：约3亿
- 当前价格0.024（FDV 24亿）= 合理，非"低估"
```

**差额处理手段**：
- 🔒 锁仓
- 🧙‍♀️ 反女巫
- 🚫 定向降低/取消某活动维度占比

**平衡三目标**：
1. 最大化单位撸毛的折算FDV
2. 保持单号收益在不FUD范围内
3. 最大化老鼠仓持有

**冲突时优先级**：
> 优先保证最有影响力账号的情绪稳定，对超大户和超小户开刀

## 三、撸毛党侧理解

### 3.1 撸毛党本质

撸毛参与者是**100%因果的POW思维**：你让我办事，你给我发钱

可套用**矿机分红盘逻辑**：
- 矿机成本 = 脚本/招小弟成本
- 电费 = 高额手续费/gas
- 查女巫 = 矿难归零

> "BTC矿工不会二级买比特币" = 撸毛党不会变成二级交易者

### 3.2 撸毛党评估框架

撸毛可参与度主要看：
- 融资额及VC阵容
- 国库收入
- 可见成本

### 3.3 项目方应对策略

| 策略 | 说明 |
|------|------|
| 前期投入PR资源 | 让VC多出面降低信任成本 |
| 提供脑补指标 | 暗示有实力发大毛 |
| 淡化显性成本 | 将成本后置到沉没成本覆水难收之后 |

## 四、空投设计公式汇总

### 4.1 总比例公式
```
空投总比例 = $空投实际预计拨比/上线时FDV + 老鼠仓硬性要求比例
```

### 4.2 首日约束公式
```
首日交易所总买盘 ≥ 预计首日均价 × (TGE解锁 + TGE实发空投)
```

### 4.3 实发空投上限
```
实发空投上限 = 首日买盘 / 预计首日价格
名义空投上限 = 实发空投上限 + 老鼠仓占比
差额 = Tokenomics公布占比 - 名义空投上限 → 需要锁仓/反女巫/降维度
```

## 五、L2的特殊问题

### 5.1 L2空投困境

L2的gas是BTC/ETH，主交易对报价也是。意味着：
- L2主币上线后没有持有逻辑（不像APT/SUI需要拆分盘打新）
- **所有二级散户持仓都可视为抛压**

> "你为什么会认为L2会慷慨给你发空投？"

### 5.2 撸毛人建议

1. 不要撸无法做拆分盘的项目币（如大多数L2）
2. 做产生真钱真交易的任务，范围广而非单个任务量大
3. 尽可能成为项目方或KOL，做能威胁项目方的人
4. 保持平常心

## 六、TGE前社区的本质

### 6.1 核心观点

> "在产生与主币有关的流动性之前，任何TGE前的'社区'事实上都没有意义"

### 6.2 历史案例

| 项目 | 空投策略 | 结果 |
|------|----------|------|
| **Sui** | 测试网不发空投，只给交易所白名单 | 全网骂SuiScam，但Cetus IDO后挽回 |
| **APT** | 测试网mint NFT就给低保 | mojito rug后生态乏力 |
| **Merlin** | 开盘慷慨分发 | 价格没维持住，照样被骂 |

**结论**：
- 发不发空投都会被骂
- 后期口碑与空投无关，只与**预期管理**和**代币利益绑定**有关

### 6.3 项目方视角

> "权力只向权力的来源负责"

项目前期做社区做数据是为了和交易所谈判做筹码。只有产生流动性绑定的社区，才是真正的社区。

## 七、操作清单

### 7.1 项目方空投设计清单

| 阶段 | 关键动作 |
|------|----------|
| **领投前** | ☐ 选择0成本数据类型 ☐ 设计可视化积分 ☐ 找大V发教程 |
| **上所前** | ☐ 增加活动维度 ☐ 模糊亲儿子关系 ☐ 内部明确阵营 |
| **TGE前** | ☐ 计算首日买盘 ☐ 倒推实发空投上限 ☐ 设计锁仓/反女巫策略 |

### 7.2 撸毛人判断清单

| 检查项 | 问题 |
|--------|------|
| 拆分盘可能 | 这个项目币能做拆分盘吗？ |
| 任务真实性 | 任务是否产生真钱真交易？ |
| 维度分散 | 我的活动是否覆盖多个维度？ |
| 退出预期 | 我能在什么时候安全退出？ |
