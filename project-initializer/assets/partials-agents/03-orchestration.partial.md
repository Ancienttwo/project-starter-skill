## Workflow Orchestration

1. Contextualize
- Read relevant files deeply before planning.
- Capture findings in `docs/research.md` for persistence.

2. Set Objectives
- Define goal, success criteria, scope boundaries, and constraints.
- Separate discoverable facts from preference decisions.

3. Parallelize Safely
- Run independent analysis tasks in parallel.
- Keep mutating changes phased and scoped.

4. Execute Plan
- Build a task checklist from `docs/plan.md`.
- Keep one active task at a time.
- Mark completed tasks immediately after verification.

5. Verify by Gate
- Run typecheck, tests, lint/build based on impact.
- On failure: pause, fix root cause, update plan, re-run checks.
